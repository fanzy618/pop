# 里程碑计划与完成定义

## 统一执行规则

每个里程碑都遵循以下闸门：

1. 实现该里程碑范围。
2. 补齐该里程碑相关测试（不含 Console UI 自动化）。
3. 执行 `go test ./...` 并全部通过。
4. 提交本地 git 后再进入下一个里程碑。

## M0 - 文档基线

### 目标

- 建立工程约束和范围共识。

### 范围

- 新增 `AGENT.md`，明确产品目标、MVP 边界和里程碑流程。

### 完成定义

- 约束可被后续里程碑直接引用执行。

## M1 - 核心代理骨架

### 目标

- 打通最小可用代理路径。

### 范围

- HTTP 代理转发。
- CONNECT 隧道转发。
- 默认动作 `DIRECT`。

### 测试要点

- host 解析与归一化单测。
- HTTP 代理集成测试。
- CONNECT 隧道集成测试。

### 完成定义

- 在无规则复杂性下，代理可稳定工作。

## M2 - 规则引擎与阻断

### 目标

- 具备按规则决策能力。

### 范围

- 域名及子域名后缀匹配。
- 多条命中时最长 pattern 优先。
- `BLOCK` 动作（默认 404）。

### 测试要点

- 规则优先级与 pattern 匹配单测。
- 未命中默认策略验证。
- BLOCK 状态码集成测试。

### 完成定义

- 规则可准确控制请求行为。

## M3 - 上游代理分流

### 目标

- 支持按规则转发到不同上游代理。

### 范围

- `PROXY(upstream_id)`。
- 上游 HTTP 代理管理与 transport 复用。

### 测试要点

- upstream 选择逻辑单测。
- A/B 分流集成测试。
- DIRECT/BLOCK 回归验证。

### 完成定义

- 同一 POP 可对不同域名执行不同上游策略。

## M4 - 运行参数与恢复

### 目标

- 实现可恢复的运行配置生命周期。

### 范围

- 启动参数按 `CLI > ENV > default` 解析。
- 去除配置文件依赖。
- 重启后从 SQLite 恢复规则与上游策略。

### 测试要点

- 参数覆盖优先级测试。
- 重启恢复集成测试。

### 完成定义

- 规则与上游变更不会因重启丢失。

## M5 - Telemetry

### 目标

- 提供实时可观测能力且内存可控。

### 范围

- 活动事件记录。
- 统计聚合（并发、请求、错误、带宽）。
- ring buffer 容量限制与 TTL 清理。

### 测试要点

- 容量上限与淘汰行为。
- TTL 过期行为。
- 统计准确性与基本并发行为。

### 完成定义

- 长期运行活动数据不无限增长，统计可读。

## M6 - Console API

### 目标

- 通过 Console API 暴露配置管理和运行观测。

### 范围

- 配置、规则、上游 CRUD 与规则重排 API。
- stats 与 activities API。
- activities SSE。

### 测试要点

- 规则 CRUD + reorder。
- stats/activities 返回验证。
- SSE 基础可用验证。

### 完成定义

- 无 UI 自动化前提下，API 层完整可用。

## 后续候选里程碑（建议）

- M7：Console 前端页面（可视化规则编辑与实时看板）。
- M8：安全增强（密码哈希、可选 token、审计日志）。
- M9：策略增强（规则分组、导入导出、灰度策略）。
