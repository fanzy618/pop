# 需求说明

## 1. 产品定位

POP（Proxy of Proxy）是一个面向个人用户的本地 HTTP 代理服务。

核心目标按优先级排序：

1. 易用性：配置清晰、行为可预期、可快速上手。
2. 可维护性：模块边界清楚、便于演进。
3. 资源友好：长期运行时内存和连接数可控。
4. 性能满足个人访问需求，不追求超大规模吞吐。

## 2. 典型使用场景

1. 用户将系统代理设置为 POP 地址，统一经过 POP。
2. 访问内部域名时，POP 直接访问（类似普通代理）。
3. 访问外部域名时，POP 可按规则转发到代理 A 或 B。
4. 对广告域名，POP 直接返回错误码（Web Console 固定为 404）。
5. 无匹配规则时，按默认行为执行（当前默认 `DIRECT`）。
6. 通过 Web Console 查看实时活动、统计数据并管理规则。
7. 重启后规则和配置仍然存在。

## 3. 功能需求

### 3.1 代理能力

- 提供 HTTP 代理端口。
- 支持普通 HTTP 请求和 `CONNECT` 隧道。

### 3.2 路由决策

- 按域名规则进行匹配。
- 规则按顺序生效（first match wins）。
- 动作类型：
  - `DIRECT`
  - `PROXY(upstream_id)`
  - `BLOCK(404)`
- 无匹配规则时，使用默认动作（当前为 `DIRECT`）。

### 3.3 规则能力

- 规则支持启用/禁用。
- 规则支持顺序调整。
- 域名模式支持：
  - 精确匹配：`example.com`
  - 子域通配：`*.example.com`
  - 主机名通配：`*ads*`

### 3.4 上游代理

- 支持配置多个上游代理。
- 上游名称 `name` 为可选字段。
- MVP 仅支持 `http://` 类型上游代理。

### 3.5 Web Console

- 使用简单用户名/密码保护（Basic Auth）。
- 可管理配置、规则、上游代理。
- 规则与上游主键由数据库自动生成。
- 创建规则时使用统一动作下拉：`DIRECT` / `BLOCK(404)` / `上游`。
- 可查看实时活动与运行期统计。

### 3.6 持久化

- 规则与配置持久化到本地文件。
- 服务重启后自动加载。

## 4. 非功能需求

### 4.1 可维护性

- 模块职责清晰：配置、规则、代理、上游、观测、控制台分层。
- 关键行为必须可测试。

### 4.2 资源约束

- 活动日志需有界（容量上限 + TTL 清理）。
- 统计为运行期聚合，不做历史持久化。
- 连接应复用并设置合理超时。

### 4.3 安全基线（MVP）

- Console API 必须鉴权。
- 明确该版本不包含复杂用户系统与细粒度权限。

## 5. MVP 边界（Out of Scope）

- SOCKS5 上游代理支持。
- UI 自动化测试（当前仅覆盖 Console API）。
- 历史统计持久化与报表系统。
- 多租户与复杂 RBAC。

## 6. 验收标准

- 设置系统代理后，请求可经 POP 成功访问。
- 指定内部域名走 DIRECT；指定外部域名分别走上游 A/B。
- 广告域名按规则返回阻断状态码。
- 未命中规则按默认动作执行。
- Console API 可进行规则与上游 CRUD 及排序。
- 可查看实时活动与基础统计。
- 重启后配置和规则保持一致。
